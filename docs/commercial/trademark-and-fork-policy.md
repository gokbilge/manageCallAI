# Trademark and Fork Policy

Last updated: 2026-06-07.

This document explains the difference between code-license rights and the
non-code commercial rights that remain controlled by maintainers.

## Code license rights

Code license rights come from the repository license.

For the current public repository, that means users and forks may use,
reproduce, and modify the code under Apache-2.0 for the covered releases.

## Brand and trademark rights

Brand and trademark rights are separate from code license rights.

Forks do not automatically receive rights to:

- the manageCallAI product name as their primary brand
- official logos
- official badges
- certification marks
- official endorsement claims

See [`TRADEMARKS.md`](../../TRADEMARKS.md).

## Support rights

Support rights come from maintainer-operated support offerings and commercial
agreements, not from the open-source code license.

Forks do not automatically receive official support or support-portal access.

## Certified build rights

Certified build rights are separate from source access.

Forks do not automatically receive:

- official certified images
- official signed releases
- official interoperability certifications
- official support-backed builds

## Commercial module access

Commercial module access is also separate from public repository access.

Forks may use the public Core, but do not automatically receive:

- private Pro modules
- private Enterprise modules
- official license portal access
- official activation or signing services
- official migration-intelligence implementations

## Summary

The code license grants software rights.
It does not automatically grant:

- trademark rights
- official support rights
- certified build rights
- access to private commercial modules
