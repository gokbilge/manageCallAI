# Audit - Runtime Safety and FreeSWITCH Integration - 2026-05-30

**Commit:** current main as of 2026-05-30
**Scope:** `apps/api/src/modules/freeswitch/`, `apps/api/src/modules/runtime/`,
`apps/api/src/modules/call-events/`, `apps/freeswitch-agent/`, `freeswitch/`,
`scripts/`
**Result:** PASS WITH FINDINGS

## Executive Summary

The runtime integration is architecturally sound: business logic stays in the
Node.js API, FreeSWITCH is not forked, Lua remains thin, and the Go agent handles
ESL event ingestion. XML generation is deterministic and XML escaping is in
place.

The findings below must be resolved before carrying production call traffic.

## Runtime Safety Principles

- **P1:** FreeSWITCH is a runtime execution boundary. It consumes artifacts
  generated by the API. It does not own business policy.
- **P2:** Values flowing from FreeSWITCH to the API are untrusted input.
- **P3:** Values flowing from the API to FreeSWITCH must be sanitized before
  embedding in FreeSWITCH application arguments.
- **P4:** Runtime bearer credentials must not appear in URLs, access logs,
  `xml_curl` logs, screenshots, or monitoring payloads.
- **P5:** Lua contains only thin execution: answer, stream, collect, bridge,
  transfer, hangup. Routing decisions come from the API.
- **P6:** IVR sessions must reach a terminal action or explicitly wait for caller
  hangup. No infinite loops or dangling sessions.
- **P7:** ESL events the API cannot process are dropped, not allowed to crash the
  agent.
- **P8:** Degraded runtime behavior must be safe. If the API is unreachable,
  active calls should fail closed with a graceful hangup, not dead air.

## Findings

### AUD-2026-05-30-001: `mod_xml_curl` runtime token is passed in query string

- **Status:** open
- **Severity:** critical
- **Location:** `freeswitch/conf/autoload_configs/xml_curl.conf.xml.example`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/12
- **Finding:** The example `xml_curl` configuration passes the runtime token in
  the URL query string. This conflicts with production-disabled query/body token
  fallback and leaks bearer credentials through FreeSWITCH and reverse-proxy
  access logs.
- **Fix:** Replace query-token examples with a supported non-query auth path for
  FreeSWITCH runtime endpoints. Prefer HTTP Basic Auth parsing for
  `/api/v1/freeswitch/*` until node-signed runtime requests are implemented.
- **Resolved:**

### AUD-2026-05-30-002: Lua bridge arguments are not FreeSWITCH-argument sanitized

- **Status:** open
- **Severity:** critical
- **Location:** `freeswitch/scripts/managecall_entry.lua`,
  `freeswitch/scripts/inbound_route.lua`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/13
- **Finding:** API-sourced values are interpolated into bridge/application
  arguments without a dedicated FreeSWITCH application-argument sanitizer.
- **Fix:** Add a sanitizer and validation layer for bridge strings, transfer
  destinations, voicemail targets, prompt paths, and runtime action arguments.
- **Resolved:**

### AUD-2026-05-30-003: ESL agent subscribes to `events plain all`

- **Status:** open
- **Severity:** critical
- **Location:** `apps/freeswitch-agent/internal/esl/client.go`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/14
- **Finding:** The ESL client subscribes to all FreeSWITCH events and filters
  later. Busy deployments can produce large volumes of irrelevant events,
  creating avoidable TCP, parse, allocation, and reconnect risk.
- **Fix:** Subscribe only to required event classes and keep unknown events as
  drop-only.
- **Resolved:**

### AUD-2026-05-30-004: `sofia::expire` is not normalized

- **Status:** open
- **Severity:** high
- **Location:** `apps/freeswitch-agent/internal/events/normalize.go`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/15
- **Finding:** Registration expiry is dropped when a phone disappears without an
  explicit unregister event.
- **Fix:** Map `sofia::register` to `registration_seen` and both
  `sofia::unregister` and `sofia::expire` to `registration_expired`.
- **Resolved:**

### AUD-2026-05-30-005: Platform health does not include FreeSWITCH reachability

- **Status:** open
- **Severity:** high
- **Location:** `apps/api/src/modules/platform/platform.service.ts`,
  `apps/freeswitch-agent/`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/16
- **Finding:** Platform health checks API and worker state but cannot report
  whether FreeSWITCH is reachable or ESL is connected.
- **Fix:** Add a FreeSWITCH agent health endpoint and include it in platform
  runtime health.
- **Resolved:**

### AUD-2026-05-30-006: Outbound originate dial number is not strictly validated

- **Status:** open
- **Severity:** high
- **Location:** `apps/freeswitch-agent/`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/17
- **Finding:** The outbound dispatcher passes dial numbers into ESL originate
  command construction without a strict E.164 or explicit internal-number
  allowlist.
- **Fix:** Validate dial numbers before command construction and mark unsafe
  requests failed.
- **Resolved:**

### AUD-2026-05-30-007: Runtime event ingest trusts body `tenant_id`

- **Status:** open
- **Severity:** high
- **Location:** `apps/api/src/modules/call-events/call-event.service.ts`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/18
- **Finding:** Runtime ingest accepts tenant identity from the request body
  without binding it to the authenticated runtime caller.
- **Fix:** Bind runtime credentials to tenant and/or FreeSWITCH node identity and
  reject mismatched tenant ids.
- **Resolved:**

### AUD-2026-05-30-008: Pattern route lookup has regex ReDoS surface

- **Status:** open
- **Severity:** medium
- **Location:** `apps/api/src/modules/freeswitch/route-lookup.repository.ts`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/19
- **Finding:** Tenant-supplied route patterns are evaluated through PostgreSQL
  regex matching. Pathological expressions can create route lookup latency or
  denial-of-service risk.
- **Fix:** Validate route pattern syntax and complexity at creation/update time,
  enforce maximum length, and document supported pattern syntax.
- **Resolved:**

### AUD-2026-05-30-009: Voicemail and transfer arguments need boundary checks

- **Status:** open
- **Severity:** medium
- **Location:** `freeswitch/scripts/managecall_entry.lua`,
  `freeswitch/scripts/inbound_route.lua`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/13
- **Finding:** Voicemail and transfer application arguments share the same
  FreeSWITCH application-argument boundary risk as bridge construction.
- **Fix:** Resolve with the sanitizer and runtime argument validation tracked in
  issue #13.
- **Resolved:**

### AUD-2026-05-30-010: ESL reconnect uses fixed delay without jitter

- **Status:** open
- **Severity:** medium
- **Location:** `apps/freeswitch-agent/internal/esl/`
- **Issue:** https://github.com/gokbilge/manageCallAI/issues/20
- **Finding:** Fixed reconnect timing can cause many agents to reconnect in
  lockstep during outage or restart.
- **Fix:** Add bounded exponential backoff with jitter and reset after stable
  connection.
- **Resolved:**
