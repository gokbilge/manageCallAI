# Code of Conduct

## Our Standard

This project should be a practical, respectful place to discuss telecom,
FreeSWITCH, IVR, automation, and operations work.

Examples of expected behavior:

- Use welcoming and inclusive language.
- Respect differing levels of telecom and software experience.
- Keep technical criticism specific, factual, and actionable.
- Assume good intent while still being direct about safety, security, and
  production-risk concerns.

Examples of unacceptable behavior:

- Harassment, insults, threats, or sustained disruption.
- Public or private abuse of project participants.
- Sharing private information without explicit permission.
- Dismissing safety or security concerns without technical justification.

## Scope

This code of conduct applies in project spaces including GitHub Issues,
Discussions, Pull Requests, project boards, and community chat spaces operated
for manageCallAI.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or discussions;
temporarily limit participation; or block participants when behavior harms the
project community or creates security, safety, or operational risk.

Report conduct issues to the repository maintainers. Security-sensitive reports
should follow the security reporting process instead of public issues.
